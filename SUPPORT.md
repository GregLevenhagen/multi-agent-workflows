# Support

## Getting Help

Use GitHub issues for:

- Reproducible bugs
- Documentation gaps
- Feature requests

Before opening an issue, check the README, existing issues, and the docs in `docs/`.

## Support Boundaries

This is a single-maintainer open-source project. Support is best-effort. The maintainer may not be able to provide:

- Real-time troubleshooting
- Consulting or implementation services
- Custom feature work on demand

## Security Issues

For sensitive security reports, follow `SECURITY.md` instead of opening a public issue.

## Contribution Path

If you already know the fix, a focused pull request with tests is the fastest path.
